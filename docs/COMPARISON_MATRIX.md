# Consultant vs Other Advisory Archetypes

| Feature | Consultant | Advisor | Spec-Engineer |
|---------|------------|---------|---------------|
| Primary Focus | Strategic engagement | Guidance | Specification |
| Output Type | Recommendations | Analysis | Formal specs |
| Stakeholder Scope | Multi-party | Single client | Technical teams |
| Evidence Depth | Comprehensive | Focused | Requirements-based |
| Typical Engagement | Project-based | Ongoing | Deliverable-based |

## When to Use Consultant

- Complex multi-stakeholder projects
- Strategic decision support
- Due diligence reviews
- Organizational assessments

## When to Use Other Archetypes

- **Advisor**: Ongoing guidance, focused analysis
- **Spec-Engineer**: Formal specifications, technical requirements
